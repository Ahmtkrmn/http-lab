2.Mimari Yapı

Proje, klasik bir katmanlı (layered) mimari izler: her katman yalnızca bir sorumluluğa sahiptir ve bir alttaki katmanı soyutlama üzerinden kullanır (SOLID — bkz. REFACTORING_DIARY.md). Katmanlar arası bağımlılık her zaman tek yönlüdür.

Routeroutes/ → Middlewareauth, log, error → Service/Utiltoken, password → Data Accessstore/, db/ → PostgreSQL

Klasör Yapısı ve Her Katmanın Görevi

http-lab/
├── src/
│   ├── app.js              # Express app'i İNŞA EDER (route/middleware kaydı) — dinlemeye başlamaz
│   ├── server.js            # app.js'i import edip PORT'ta dinlemeye başlatan TEK yer
│   ├── db/
│   │   └── prisma.js        # Uygulama genelinde TEK paylaşılan PrismaClient singleton'ı (DIP)
│   ├── middleware/

│   │   ├── requestLogger.js    # Her isteği method/path/status/süre ile loglar
│   │   └── errorHandler.js     # Zincirin EN SONUNDAKİ global hata yakalayıcı
│   ├── routes/
│   │   ├── auth.js          # /register, /login — orkestrasyon yapar, iş mantığı barındırmaz

│       └── passwordService.js  # bcrypt hash/compare — Express'ten bağımsız, saf fonksiyonlar
├── prisma/
│   ├── schema.prisma        # Veri modeli (User, Item, Category) — tek gerçek kaynak
│   └── migrations/           # Versiyon kontrollü şema geçmişi
├── tests/
│   ├── unit/                # DB gerektirmez, mock'lanmış bağımlılıklar
│   └── integration/         # Gerçek PostgreSQL'e karşı, Supertest ile uçtan uca
└── .github/workflows/       # ci.yml + deploy.yml (bkz. Kritik Konfigürasyonlar)

Neden Bu Yapı? (Mimari Kararların Özeti)

app.js / server.js ayrımı
"Neyi inşa ediyorum" ile "nasıl çalıştırıyorum" birbirinden ayrıldı — app.js'i test dosyalarından require etmek gerçek bir ağ portu açmıyor (Supertest bunu kullanır).
Tek Prisma Singleton
db/prisma.js tüm modüllerin paylaştığı tek bağlantı havuzunu üretir (DIP) — eskiden iki ayrı bağlantı havuzu açılıyordu, bu kaldırıldı.
İki Katmanlı Auth
authenticateToken (kimlik) → requireRole (yetki) sırasıyla çalışır; ownership kontrolü ise route seviyesinde (items.js içinde) ayrıca yapılır.
Docker-first Yerel Geliştirme
docker compose up --build asıl çalıştırma yöntemi kabul edildi — production'daki (Render) container davranışıyla birebir eşleşir.

4.Veri Akışı

4.1 — Yazma İsteği: POST /api/items

İstemciBearer token + JSON body → app.jsexpress.json() + requestLogger → items.js router → authenticateToken → requireRole('EDITOR') → route handlervalidasyon → itemsDb.create() → Prisma Client → PostgreSQL

Hata durumunda (örn. geçersiz categoryId), akış next(err) ile doğrudan errorHandler.js'e atlar — ara katmanlar hata formatlamayla ilgilenmez.

4.2 — Kimlik Doğrulama Akışı: POST /api/auth/login

İstemciemail + password → auth.jsloginLimiter → prisma.user.findUnique → passwordServicecomparePassword → tokenServicegenerate*Token → prisma.user.updaterefreshToken kaydı → İstemciaccessToken + refreshToken

Sonraki tüm isteklerde istemci accessToken'ı Authorization: Bearer <token> header'ında gönderir; authMiddleware.js bunu tokenService.verifyAccessToken ile doğrulayıp req.user = { userId, role } olarak zincire enjekte eder.

4.3 — Deployment Akışı (CI/CD)

git pushmain'e merge → ci.ymllint + test + build (paralel) → deploy.ymlRender deploy hook (curl -sf) → RenderDockerfile build + çalıştır → smoke testGET /health → 200 bekle

Render tarafında konteyner ayağa kalkarken Dockerfile'daki CMD ["sh","-c","npx prisma migrate deploy && npm start"] otomatik olarak bekleyen migration'ları uygular, sonra sunucuyu process.env.PORT (Render'ın enjekte ettiği) üzerinde başlatır.

Health Check'in Çift Rolü

GET /health (src/app.js) hem CD pipeline'ının smoke test'i hem de (Week 8'de) UptimeRobot gibi bir dış izleme aracının hedefi olacak. prisma.$queryRaw\`SELECT 1\` ile gerçek DB bağlantısını test eder; başarısız olursa 503 + db:"disconnected" döner.

5.Kritik Konfigürasyonlar

Uygulamanın ayakta kalmasını, doğru ortamda doğru şekilde çalışmasını sağlayan temel dosyalar:

DosyaGöreviDikkat Edilmesi Gereken
.env Yerel (gerçek/dev) ortam değişkenleri: DATABASE_URL, JWT_ACCESS_SECRET, JWT_REFRESH_SECRET, PORT. .gitignore'da — asla commit edilmez. DATABASE_URL host'u docker compose up ile çalıştırmak için db olmalı, localhost değil.
.env.test Test ortamı değişkenleri — ayrı bir http_lab_test veritabanını gösterir. Her npm test çalıştığında bu veritabanı tamamen temizlenir (resetDb.js). Yoksa jest.setup.js açıklayıcı bir hata fırlatır.
prisma/schema.prisma Veri modelinin TEK gerçek kaynağı; prisma migrate dev/deploy buradan SQL migration üretir. Şema değişince mutlaka yeni bir migration dosyası (prisma/migrations/) oluşturulmalı, elle SQL yazılmamalı.
prisma.config.ts Prisma 7'nin yeni config formatı — şema yolu, migration yolu, seed komutu ve DATABASE_URL'in nereden okunacağını tanımlar. import "dotenv/config" içerdiği için .env'i otomatik yükler.
package.json scripts (test/lint/dev zinciri), jest config (coverage eşikleri: %80/%70/%80/%80), postinstall (prisma generate). pretest hook'u her npm test'ten önce db:migrate:test'i otomatik çalıştırır — elle migration gerekmez.
Dockerfile Multi-stage build (builder/runner); production image'ı non-root node kullanıcısıyla, minimum dosyayla çalıştırır. CMD, container her başladığında prisma migrate deploy'u otomatik uygular — production'da genelde ayrı bir adım olur, burada laboratuvar amaçlı birleştirilmiş.
docker-compose.yml Yerel "tam stack" ortamı: app + db + adminer. db servisinin host portu kasıtlı olarak KAPALI (güvenlik) — sadece Docker iç ağından erişilebilir.
eslint.config.js Statik kod analizi kuralları (flat config formatı, ESLint 9+). no-unused-vars kasıtlı olarak gevşetilmiş (caughtErrors:'none') — projenin catch(err) stiliyle çakışmasın diye.
.github/workflows/ci.yml lint + test (gerçek Postgres service container ile) + build job'ları; her push/pull_request'ta çalışır. Gerekli secret: TEST_DATABASE_URL. .env.test dosyası CI içinde bu secret'tan dinamik olarak üretiliyor.
.github/workflows/deploy.yml main'e merge sonrası Render'a deploy tetikler ve /health'i smoke test eder. Gerekli secret: RENDER_DEPLOY_HOOK. Opsiyonel variable: RENDER_SERVICE_URL (yoksa sabit URL'e düşer).
